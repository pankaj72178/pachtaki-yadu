                         🚀 HOW IT WORKS
═══════════════════════════════════════════════════════════════════════════════

USER FLOW:
1. User opens index.html in browser
2. User fills Citizen Feedback form
3. User clicks "SUBMIT REPORT"
4. Form validates all fields
5. Form sends data to backend API
6. Backend receives POST request
7. Backend validates data
8. Backend saves to complaints.json
9. Backend sends success response
10. Frontend shows green success message
11. Form clears automatically

DATA FLOW:
Browser Form
    ↓
JavaScript Handler
    ↓
Fetch API (POST)
    ↓
http://localhost:3000/api/complaints
    ↓
Express Server
    ↓
Input Validation
    ↓
JSON Storage
    ↓
Success Response
    ↓
Display Message & Clear Form

                         📊 API DOCUMENTATION
═══════════════════════════════════════════════════════════════════════════════

1. HEALTH CHECK
   Endpoint: GET /api/health
   Purpose: Check if server is online
   Response:
   {
     "status": "Server is running!",
     "timestamp": "2026-05-08T10:00:00.000Z"
   }

2. SUBMIT COMPLAINT
   Endpoint: POST /api/complaints
   Purpose: Submit new complaint
   Request Body:
   {
     "fullName": "John Doe",
     "wardNumber": "Ward 1",
     "issue": "Road needs repair"
   }
   Response:
   {
     "success": true,
     "message": "Complaint submitted successfully!",
     "complaint": {
       "id": 1715209600000,
       "fullName": "John Doe",
       "wardNumber": "Ward 1",
       "issue": "Road needs repair",
       "status": "Pending",
       "createdAt": "2026-05-08T10:30:45.123Z",
       "updatedAt": "2026-05-08T10:30:45.123Z"
     }
   }

3. GET ALL COMPLAINTS
   Endpoint: GET /api/complaints
   Purpose: Retrieve all complaints
   Response: Array of complaint objects

4. GET SPECIFIC COMPLAINT
   Endpoint: GET /api/complaints/:id
   Purpose: Get one complaint by ID
   Response: Single complaint object
